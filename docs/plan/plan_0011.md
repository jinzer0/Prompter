# Phase 11 최종 수정 계획: 클립보드 빠른 캡처

## 목표

Phase 11은 외부 앱에서 사용자가 명시적으로 복사한 클립보드 텍스트를 Prompter의
Prompt Compiler `Original request`로 가져오는 빠른 진입점을 추가한다. 이 단계는
프롬프트 실행 기능이 아니며, 자동 LLM 호출이나 자동 저장 기능도 아니다.

최종 방향은 “비용 없는 명시적 클립보드 가져오기”다. 사용자는 버튼, 앱 메뉴, 앱이
포커스된 상태의 내부 단축키로 클립보드 텍스트를 가져온 뒤 직접 analyze 또는 compile을
실행한다.

## 현재 코드 기준 결정

- `window.prompter.clipboard.copyText(input)`는 이미 존재하므로 재사용한다.
- `readText()`만 기존 clipboard bridge에 최소 추가한다.
- `electron/app-menu.ts`의 `MENU_ACTIONS`, `MENU_ACTION_CHANNEL`, preload Zod 검증,
  `window.prompter.menu.onAction(callback)` 패턴을 재사용한다.
- 별도 `appEvents` bridge를 만들지 않는다.
- `renderer/src/app.tsx`의 기존 `data-menu-action-target` 클릭/포커스 라우팅 패턴을
  따른다.
- `usePromptCompilerPanel`이 compiler draft, analysis, answers, compiled,
  editablePrompt, suggested tags, message를 소유하므로 quick capture는 단순 `setDraft`가
  아니라 hook-level import/reset 함수를 통해서만 수행한다.
- `useCompilerDefaults`가 이미 기본 scenario와 target agent를 draft에 반영하므로 quick
  capture는 현재 draft 값을 보존한다. 별도 settings를 추가하지 않는다.

## 최종 범위

| 항목 | 결정 |
|---|---|
| Clipboard read API | `window.prompter.clipboard.readText()`를 추가한다. payload는 없다. |
| 반환값 | `{ text: string; isEmpty: boolean; length: number }` |
| Native read | Electron main/native service에서 `clipboard.readText()`를 호출한다. renderer는 직접 접근하지 않는다. |
| UI 버튼 | Prompt Compiler 영역에 `Import from Clipboard` 버튼을 추가한다. |
| 앱 메뉴 | File 메뉴에 `Quick Capture from Clipboard`를 추가한다. |
| 내부 단축키 | Electron menu accelerator `CmdOrCtrl+Shift+V`를 사용한다. OS 전역 단축키가 아니다. |
| 입력 위치 | clipboard text는 `draft.originalInput`에 들어간다. |
| 텍스트 처리 | 원문을 그대로 가져온다. trim, 빈 줄 정리, 제어문자 제거, truncation을 하지 않는다. |
| 빈 클립보드 | draft를 바꾸지 않고 명확한 메시지를 표시한다. |
| 긴 클립보드 | `length >= 20000`이면 원문 그대로 가져오고 비용/처리 시간 경고를 표시한다. |
| 기존 입력 덮어쓰기 | 현재 original input이 비어 있지 않고 클립보드 텍스트와 다르면 in-app 확인을 요구한다. |
| 같은 텍스트 | 확인 없이 textarea focus와 상태 메시지만 처리한다. |
| 상태 초기화 | 실제 import 시 stale analysis, answers, compiled, editablePrompt, suggested tags를 초기화한다. |
| 기본값 | 현재 draft의 scenario와 targetAgent를 보존한다. |
| 저장 | 사용자가 명시적으로 저장하기 전까지 DB, PromptAsset, PromptVersion, search index에 저장하지 않는다. |
| LLM 호출 | import만으로 analyze/compile/OpenAI 호출을 실행하지 않는다. |

## 비범위

| 제외 항목 | 이유 |
|---|---|
| OS 전역 `globalShortcut` | privacy model, unregister lifecycle, 실패 UX, settings, accelerator 검증, Electron 테스트가 필요하므로 Phase 12+로 연기한다. |
| quick capture settings | MVP에는 불필요하며 schema/defaults/form/persistence/UI 범위를 크게 늘린다. |
| `window.prompter.shortcuts.*` | 전역 단축키가 없으므로 만들지 않는다. |
| `window.prompter.appEvents.*` | 기존 `menu.onAction`으로 충분하다. |
| clipboard monitoring | 사용자의 명시적 액션 없이 민감 데이터를 읽을 수 있으므로 제외한다. |
| 외부 앱 selected text 강제 읽기 | privacy boundary를 넘으므로 제외한다. |
| 텍스트 cleanup 옵션 | 코드, Markdown, log, diff, whitespace-sensitive prompt를 훼손할 수 있으므로 제외한다. |
| quick capture draft persistence | capture-adjacent data 저장 위험이 있으므로 제외한다. |
| prompt execution | 계속 제외한다. |
| Codex/Claude Code/Cursor 실행 | 계속 제외한다. |
| prompt run/result 저장 | `prompt_runs`, `agent_runs`, `execution_results`, `validation_results`, `run_logs`를 만들지 않는다. |
| DB schema/migration 변경 | Phase 11에는 persistence 변경이 필요 없다. |

## draft_0011에서 삭제/연기/변경된 내용

### 삭제

- `window.prompter.appEvents.onQuickCapture(callback)`
- `window.prompter.appEvents.off`
- `window.prompter.shortcuts.registerGlobalQuickCapture(input)`
- `window.prompter.shortcuts.unregisterGlobalQuickCapture()`
- `window.prompter.shortcuts.getGlobalQuickCaptureStatus()`
- `quick_capture_trim_clipboard_text`
- `quick_capture_warn_long_text`
- `quick_capture_long_text_threshold`
- `quick_capture_global_shortcut_enabled`
- `quick_capture_global_shortcut`
- `QuickCaptureSettingsSchema`
- `RegisterGlobalShortcutInputSchema`
- Settings table/defaults 변경
- 자동 trim, 빈 줄 정리, control character cleanup
- “원문 그대로/가벼운 정리” 옵션
- quick capture draft sessionStorage/local persistence
- 앱 종료 시 unsaved quick capture 경고

### 연기

- OS 전역 단축키
- 사용자 설정 가능한 long-text threshold
- 사용자 설정 가능한 cleanup/trim 옵션
- clipboard history
- background clipboard watch
- 외부 앱 선택 영역 읽기

### 변경

- draft의 “button 또는 shortcut”은 “버튼 + 앱 메뉴 accelerator”로 확정한다.
- append 옵션은 구현하지 않고 overwrite confirmation만 구현한다.
- 자동 trim 대신 원문 그대로 import한다.
- settings 기반 threshold 대신 renderer/hook 상수 `20000`을 사용한다.
- 별도 app event API 대신 기존 menu action channel을 재사용한다.
- global shortcut은 optional이 아니라 Phase 11 금지/연기 항목으로 명시한다.
- “default scenario/default target agent 적용”은 새로 settings를 읽는 것이 아니라 현재 draft에 이미 반영된 값을 보존하는 것으로 정의한다.

## 강화된 요구사항

- import는 `promptCompiler.analyze`를 호출하지 않는다.
- import는 `promptCompiler.compile`을 호출하지 않는다.
- import는 prompt create/version create/settings set/export save/search rebuild/secrets API를 호출하지 않는다.
- clipboard 내용을 main/preload/renderer/test/log에 출력하지 않는다.
- renderer는 Electron, `ipcRenderer`, Node, filesystem, `process`, `navigator.clipboard`를 직접 사용하지 않는다.
- 실제 import는 반드시 hook-level reset/import 함수 하나를 통해 수행한다.
- clipboard text는 원문 그대로 유지한다.
- clipboard text는 사용자가 명시적으로 저장/내보내기/복사하기 전까지 renderer draft state에만 존재한다.

## 구현 순서와 TDD 게이트

### 1. Clipboard read contract 테스트 추가

작업:

- `readText()` channel, no-payload request, response shape를 실패 테스트로 먼저 정의한다.
- native clipboard read service가 non-empty/empty 결과를 반환하는지 테스트한다.
- bridge response schema가 malformed result를 거부하는지 테스트한다.

대상 파일:

- `tests/phase8-export-contract.test.ts`
- `tests/electron-contract.test.ts` 필요 시 보강

완료 조건:

- 구현 전에는 `readText` 부재로 실패한다.
- 반환 shape는 `{ text, isEmpty, length }`로 고정된다.

검증:

```bash
npm run native:node && npx vitest run tests/phase8-export-contract.test.ts tests/electron-contract.test.ts
```

### 2. Clipboard read IPC/native path 구현

작업:

- `electron/ipc-contract.ts`에 `readText` channel과 response schema를 추가한다.
- `electron/bridge-types.ts`, `electron/bridge.ts`에 `clipboard.readText()`를 추가한다.
- `electron/ipc-handlers.ts`에 no-payload 검증 후 `services.readText()` 호출을 추가한다.
- `electron/prompt-export-native.ts` 또는 기존 clipboard native service 위치에 `readText`를 추가한다.
- `electron/main.ts`에서 `clipboard.readText()` dependency를 주입한다.

완료 조건:

- `window.prompter.clipboard.readText()`가 typed bridge로 동작한다.
- renderer에 native API가 노출되지 않는다.
- `copyText` 동작은 그대로 유지된다.

### 3. Quick Capture menu contract 테스트 추가

작업:

- `MENU_ACTIONS`에 `quickCaptureFromClipboard`가 포함되는 실패 테스트를 추가한다.
- File 메뉴에 `Quick Capture from Clipboard`와 accelerator `CmdOrCtrl+Shift+V`가 있는지 테스트한다.
- 메뉴 클릭이 `quickCaptureFromClipboard` action을 보내는지 테스트한다.
- renderer target mapping이 `quick-capture-from-clipboard`를 기대하도록 테스트한다.

대상 파일:

- `tests/electron-contract.test.ts`

완료 조건:

- 구현 전에는 action/menu item 부재로 실패한다.
- 테스트는 OS 전역 shortcut이 아니라 Electron menu accelerator만 요구한다.

### 4. Compiler import/reset hook 구현

작업:

- `usePromptCompilerPanel`에 `QUICK_CAPTURE_LONG_TEXT_THRESHOLD = 20000` 상수를 둔다.
- `pendingClipboardImport`, `isReadingClipboard` 등 최소 상태를 추가한다.
- 실제 import 전 stale derived state를 초기화하는 private reset 함수를 추가한다.
- `importFromClipboard()`, `confirmClipboardImport()`, `cancelClipboardImport()`를 추가한다.
- 기존 draft의 `scenario`, `targetAgent`, 기타 explicit form fields를 보존하고 `originalInput`만 교체한다.
- 빈 클립보드는 draft와 derived state를 바꾸지 않는다.
- overwrite confirmation이 필요한 경우 confirm 전까지 어떤 상태도 변경하지 않는다.

초기화 대상:

- analysis
- clarification answers
- compiled result
- editable prompt
- suggested tag selection
- generated suggested tags
- stale success/warning message

완료 조건:

- import action은 clipboard read만 수행한다.
- import action은 LLM, 저장, export, search rebuild, settings, secrets를 호출하지 않는다.
- 실제 import 시 stale compiled output이 copy/save 가능한 상태로 남지 않는다.

### 5. Import UI, confirmation, warning, focus 구현

작업:

- Prompt Compiler action area에 `Import from Clipboard` 버튼을 추가한다.
- 버튼에 `data-menu-action-target="quick-capture-from-clipboard"`를 붙인다.
- 읽는 중에는 버튼을 disabled 처리하고 `Reading Clipboard...` 상태를 보여준다.
- overwrite confirmation UI를 in-app으로 렌더링한다. 브라우저 `confirm()`은 사용하지 않는다.
- confirm 버튼: `Replace original request`
- cancel 버튼: `Cancel import`
- 긴 텍스트 경고를 표시한다.
- 성공 후 Original request textarea에 focus한다.
- 필요하면 `PromptCompilerForm`과 `Textarea` wrapper에 ref forwarding을 최소 변경으로 추가한다.

UI 문구:

- `Import from Clipboard`
- `Clipboard does not contain text to import.`
- `Clipboard text imported.`
- `Clipboard text is already in the original request.`
- `Replace the current original request with clipboard text?`
- `Imported clipboard text is very long; review it before analysis or compile.`

완료 조건:

- 버튼은 keyboard reachable하고 accessible name이 명확하다.
- focus는 실제 import 후에만 이동한다.
- 기존 compact dark three-panel UI와 design token을 유지한다.

### 7. Phase 11 자동 테스트 추가

새 테스트 후보:

- `tests/phase11-quick-capture-ui.test.ts`

필수 테스트:

- 클립보드 텍스트가 있을 때 import button이 original input을 정확히 채운다.
- 클립보드가 비어 있으면 메시지를 표시하고 draft를 바꾸지 않는다.

- 20,000자 이상 텍스트는 원문 전체가 들어가고 경고를 표시한다.
- import만으로 LLM analyze/compile이 실행되지 않는다.
- import만으로 PromptAsset/PromptVersion이 생성되지 않는다.
- 메뉴 또는 accelerator 경로가 버튼과 같은 import flow를 실행한다.
- renderer source에 `ipcRenderer`, `from "electron"`, `node:fs`, `node:path`, `node:process`, `navigator.clipboard`, `globalShortcut`, `appEvents`, `window.prompter.shortcuts`가 없다.
- source/migration에 `prompt_runs`, `agent_runs`, `execution_results`, `validation_results`, `run_logs`가 추가되지 않는다.

완료 조건:

- happy path, empty, overwrite confirm/cancel, long warning, stale reset, menu route, no-auto-save, no-auto-LLM, forbidden surface를 모두 검증한다.

| no auto LLM | OpenAI key 없이 import | key warning/analyze/compile이 자동 시작되지 않음 |
| no auto persistence | import 후 prompt list 확인 | 명시적 저장 전 asset/version 생성 없음 |
| log privacy | devtools/terminal 확인 | clipboard 내용이 출력되지 않음 |

## 완료 기준

- 사용자가 버튼, File 메뉴, 앱 포커스 내부 accelerator로 clipboard text를 Original request에 가져올 수 있다.
- 가져온 텍스트는 원문 그대로이며 자동 LLM 호출, 저장, export, search indexing, logging, persistence가 없다.
- 다른 clipboard text를 가져온 뒤 이전 analysis/compiled result가 보이거나 copy/save 가능한 상태로 남지 않는다.
- 기존 menu/bridge 아키텍처를 재사용하고 broad bridge surface를 추가하지 않는다.
- targeted tests와 전체 validation이 통과한다.
- 최종 보고에서 draft 대비 수정/추가/삭제/연기 항목을 명확히 설명한다.

## 구현 후 보고 형식

구현 worker는 완료 후 다음을 보고한다.

- 변경 요약
- 변경 파일 목록
- 추가된 설정값: Phase 11에서는 없음
- clipboard read/import 데이터 흐름
- 앱 메뉴와 accelerator 연결 방식
- clipboard 내용이 자동 저장/로그/LLM 호출되지 않도록 한 방식
- 실행한 검증 명령과 결과
- 수동 테스트 절차
- 구현하지 않고 연기한 기능: global shortcut, quick capture settings, cleanup options, draft persistence, clipboard history/monitoring

## 출처

이 계획은 `/hyperplan` adversarial team 검토 후 plan agent 산출물을 기반으로 작성했다. Team round의 핵심 결론은 Phase 11을 기존 menu bus와 최소 `clipboard.readText()` 기반의 비용 없는 explicit import path로 축소하고, global shortcut/settings/cleanup/draft persistence/history/monitoring은 연기하는 것이다.
