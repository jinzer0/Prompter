# Phase 16 Final Plan

Provenance: generated by the /hyperplan adversarial team and sequenced by the plan agent.

## Context

Target write path: `docs/plan/plan_0016.md`.

Phase 16 adds explicit local backup export, backup validation, import preview, and safe duplicate import for Prompter data. It preserves Phase 11-15 guardrails and does not add prompt execution, run-result storage, external app control, repo scanning, global shortcuts, quick capture settings, cloud sync, background backup, remote upload, backup encryption, or settings import/export.

Current codebase facts to preserve:

- `prompt_asset_lineage` table does not exist and must not be added.
- Prompt asset lineage is stored on `prompt_assets` via `parent_prompt_id`, `parent_prompt_version_id`, and `derivation_type`.
- `prompt_templates` exists.
- `prompt_quality_reviews` exists.
- `MENU_ACTION_CHANNEL` and `menu.onAction` exist and must be reused for menu actions.
- `rebuildSearchIndex()` exists and must be used or matched by a transaction-safe search update.
- Renderer code must not import Node, Electron internals, SQLite, Drizzle, filesystem APIs, paths, or environment variables directly.

## Fixed Decisions

1. MVP excludes settings export/import for v1. Non-secret settings are out of scope, except tests must prove secrets/status are not exported.
2. Export, validate, and import use explicit backup IPC. Validation opens/reads the file in main and stores a temporary import session. Import uses session id. Renderer never receives file path or parsed backup payload.
3. Use strict `BackupEnvelope` v1 with discriminated `backupType`: `full`, `project`, `prompt_assets`, `prompt_templates`, `harness_templates`.
4. Import strategy is `safe_duplicate` only. No overwrite, merge by id/name, row-level selective import, cloud sync, encryption, scheduler, background backup, remote upload, prompt execution, or run-result storage.
5. Use a complete relationship resolution table, not naive new-ID mapping: `create(new UUID)`, `reuse(existing tag ID)`, `redirect(explicit destination project for prompt asset packs)`, or `drop(null + warning)`. Precompute mappings before writes.
6. `prompt_assets` imports require explicit `destinationProjectId` because the renderer/library is project-scoped. Do not silently target selected project or create invisible unscoped assets.
7. `project` and `full` imports create imported projects with deterministic conflict renaming. Project backup includes only internal relationships; external lineage/source refs become warnings and nullable/dropped references.
8. Tag name conflicts reuse existing tags by default for v1. Do not add tag rename/duplicate strategy in v1.
9. Preserve `prompt_quality_reviews` historical `snapshot` payloads byte-equivalent/semantically exact. Remap only relational columns such as `prompt_version_id`. Do not rewrite nested snapshot IDs unless they are formal relational fields in current schema.
10. Import must be transactional. No partial import. FTS/search must be updated or rebuilt consistently with imported versions. If existing search rebuild is not transaction-safe, implementation must fail or warn honestly and must not pretend full success.
11. Import preview is read-only and must show item counts, conflicts, warnings, secret exclusion, plaintext warning, and a consequence ledger explaining import adds copies and never overwrites.
12. Import requires explicit confirmation, but confirmation is UI state only. Do not trust renderer `confirmed: true` as a security boundary. Import service validates session freshness and preview fingerprint/revision.
13. No import auto-select/current draft mutation. Optional post-success `View imported project` action is allowed.
14. Add hard size/input bounds for backup files and Zod arrays/strings where practical. Too-large files fail or warn before parse/DB writes.
15. Temporary import sessions are main-process memory only, expire, are consumed/invalidated after import/cancel/failure where appropriate, and disappear on restart.
16. Add guardrail tests for forbidden run/execution tables/APIs, renderer Node/Electron/DB imports, `repo_path` reads/scans, `appEvents`/`shortcuts`/`globalShortcut`, `quick_capture` settings, backup encryption, scheduler/background/remote upload, and secret/status export.

Additional fixed policies:

- Backup files are plaintext JSON. UI warns that prompt, template, context, and review content may be sensitive.
- Do not persist a checksum field in v1. Use internal SHA-256 only for import-session fingerprinting.
- Maximum backup file size: `50 MiB`.
- Import session TTL: `15 minutes`.
- Deterministic imported-name conflict format: `{name} Imported`, `{name} Imported 2`, `{name} Imported 3`.
- `repo_path` may remain metadata in project context profiles, but Phase 16 must never read, stat, validate, scan, or git-inspect it.

## Backup Envelope

`BackupEnvelope` v1 is strict and discriminated by `backupType`.

Required top-level fields:

- `schemaVersion: 1`
- `appName: "Prompter"`
- `backupType: "full" | "project" | "prompt_assets" | "prompt_templates" | "harness_templates"`
- `exportedAt: number`
- `exportedByAppVersion?: string`
- `metadata: BackupMetadata`
- `data: BackupDataByType`

Required metadata fields:

- `itemCounts`
- `sourceSummary`
- `excludesSecrets: true`
- `excludesSecretStatus: true`
- `includesSettings: false`
- `plaintext: true`
- `schemaVersion: 1`

Data by backup type:

- `full`: projects, prompt assets, prompt versions, tags, prompt tag links, harness templates, project context profiles, prompt templates, prompt quality reviews.
- `project`: one project plus its prompt assets, prompt versions, tags, prompt tag links, project context profiles, internal prompt templates, internal quality reviews, and internal lineage/source relationships.
- `prompt_assets`: selected prompt assets, their prompt versions, prompt tag links, referenced tags, quality reviews, and lineage metadata for selected/internal relationships.
- `prompt_templates`: selected or all prompt templates, with nullable source references.
- `harness_templates`: selected or all harness templates.

Forbidden envelope/data keys:

- `settings`
- `openAIKey`
- `apiKey`
- `secret`
- `token`
- `accessToken`
- `refreshToken`
- `safeStorage`
- `ciphertext`
- `keychain`
- `prompt_runs`
- `agent_runs`
- `execution_results`
- `validation_results`
- `run_logs`

Validation policy:

- Unknown top-level keys are rejected.
- Unknown entity fields are rejected unless explicitly documented.
- Forbidden keys are rejected at envelope, metadata, and data collection levels.
- Prompt body content is not automatically redacted because that corrupts user data; UI warns that user-authored secrets inside prompt content remain plaintext.

## Import Resolution

Mappings are precomputed during validation/session creation and revalidated immediately before import writes.

| Source | Resolution | Notes |
|---|---|---|
| `projects.id` in `full`/`project` | `create(new UUID)` | Name uses deterministic imported suffix |
| `prompt_assets.id` in `full`/`project` | `create(new UUID)` | `project_id` maps to imported project |
| `prompt_assets.id` in `prompt_assets` | `create(new UUID)` | `project_id` redirects to explicit `destinationProjectId` |
| `prompt_assets.current_version_id` | mapped imported version ID | Must point to imported version for that asset |
| `prompt_assets.parent_prompt_id` | mapped imported asset ID or `null` | External/missing parent becomes warning |
| `prompt_assets.parent_prompt_version_id` | mapped imported version ID or `null` | External/missing parent version becomes warning |
| `prompt_assets.derivation_type` | preserve valid current enum or `null` | Only `duplicate`/`derived` |
| `prompt_versions.id` | `create(new UUID)` | `prompt_asset_id` maps to imported asset |
| `tags.id` | `reuse(existing tag ID)` by name or `create(new UUID)` | Existing name reuse is mandatory |
| `prompt_tags.prompt_asset_id` | mapped imported asset ID | Drop with warning only if source cannot be imported |
| `prompt_tags.tag_id` | reused or mapped tag ID | No duplicate tag names |
| `project_context_profiles.id` | `create(new UUID)` | `project_id` maps to imported project; `repo_path` is metadata only |
| `prompt_templates.id` | `create(new UUID)` | Name uses deterministic imported suffix |
| `prompt_templates.source_prompt_asset_id` | mapped imported asset ID or `null` | External/missing source becomes warning |
| `prompt_templates.source_prompt_version_id` | mapped imported version ID or `null` | External/missing source becomes warning |
| `harness_templates.id` | `create(new UUID)` | Name uses deterministic imported suffix |
| `prompt_quality_reviews.id` | `create(new UUID)` | `prompt_version_id` maps to imported version |
| `prompt_quality_reviews.snapshot` | preserve exactly | Do not rewrite nested IDs |
| Unsupported formal relationship | `drop(null + warning)` | Never synthesize fake IDs |

## IPC Surface

Add only typed backup IPC through the central contract, bridge, handlers, and services.

Renderer-facing namespace:

- `window.prompter.backup.exportFullBackup(input)`
- `window.prompter.backup.exportProjectBackup(input)`
- `window.prompter.backup.exportPromptAssetsBackup(input)`
- `window.prompter.backup.exportPromptTemplatesPack(input)`
- `window.prompter.backup.exportHarnessTemplatesPack(input)`
- `window.prompter.backup.validateBackupFile()`
- `window.prompter.backup.importBackup(input)`
- `window.prompter.backup.cancelImportSession(input)`

Input rules:

- Export inputs do not include settings options.
- Prompt asset export accepts non-empty `promptAssetIds`.
- Prompt template export accepts non-empty `promptTemplateIds` or `includeAll: true`.
- Harness template export accepts non-empty `harnessTemplateIds` or `includeAllUserTemplates: true`.
- Validation has no renderer path payload; main opens the file dialog.
- Import input includes `importSessionId`, `previewFingerprint`, `previewRevision`, `strategy: "safe_duplicate"`, and `destinationProjectId` when backup type is `prompt_assets`.
- Cancel input includes `importSessionId`.

Result rules:

- Export results return `cancelled`, `backupType`, `itemCounts`, and message. They do not return file path.
- Validation returns `cancelled` or preview. Preview includes `importSessionId`, fingerprint, revision, counts, conflicts, warnings, secret/status exclusion, plaintext warning, consequence ledger, required destination flags, and expiry. It does not include file path or parsed payload.
- Import returns counts, created IDs for optional navigation, warnings, and search-index status. It does not mutate current renderer selection.
